## 4. Compiler vs Interpreter (Core Concept)

Both **compiler** and **interpreter** translate:

> **Source Code â†’ Machine Code (Binary)**  
(Computers understand only 0s and 1s)

---

### ğŸ”¹ Compiler Characteristics

- Translates the **entire program at once**
- Displays **all errors together**
- Program runs **only after all errors are fixed**
- Faster execution after compilation  

**Example:** C Language  

---

### ğŸ”¹ Interpreter Characteristics

- Translates **line by line**
- Executes one line at a time
- Stops at the **first error**
- Execution resumes only after fixing the error  

**Example:** Python  

---

### ğŸ”‘ Key Difference Summary

| Compiler | Interpreter |
|--------|------------|
| Converts entire code | Converts line by line |
| Shows all errors together | Shows errors one at a time |
| Faster execution | Slower but precise |
| Not ideal for testing | Ideal for scripting & testing |

---
